# Database Usage & Schema Mapping: Skylink Bundlefasta

This document details the database schema, table mapping, Row Level Security (RLS) policies, and unused tables in the Supabase project.

## 🗄️ Database Tables Mapping

The Supabase project contains 19 tables. The following is their purpose and classification:

### 🔑 Critical Tables (Core Operations)
| Table | Description | Key Columns |
| :--- | :--- | :--- |
| `public.me` | Admin profile (Stores the single admin account metadata) | `id` (uuid), `email` (text), `password_hash` (text), `keyword_hash` (text) |
| `public.mpesa_credentials` | Stores credentials for Daraja API integration | `id` (uuid), `consumer_key` (text), `consumer_secret` (text), `passkey` (text) |
| `public.transactions` | Log of all M-Pesa paybill/express transactions | `id` (uuid), `mpesa_reference` (text), `phone_number` (text), `amount` (numeric), `status` (text) |
| `public.reversals` | Initiated and completed M-Pesa reversals | `id` (uuid), `original_transaction_id` (uuid), `amount` (numeric), `status` (text), `reason` (text) |
| `public.customers` | Unique customer records mapped from transactions | `id` (uuid), `phone_number` (text), `full_name` (text), `status` (text) |
| `public.audit_logs` | Logs of administrator activity (login, settings change, reversals) | `id` (uuid), `admin_id` (uuid), `action` (text), `ip_address` (text) |

### 🛠️ Auxiliary Tables (System & Logs)
| Table | Description |
| :--- | :--- |
| `public.mpesa_stk_requests` | Log of STK push requests generated by the dashboard |
| `public.payment_links` | Dynamic checkouts generated for payment links |
| `public.payouts` | Outflows/disbursements generated from the paybill |
| `public.reconciliation_runs` | Logs of automated matching jobs |
| `public.stk_push_logs` | Safaricom callback logs for STK pushes |
| `public.system_logs` | App errors and system status reports |
| `public.system_settings` | Operational config (transaction limits, notification endpoints) |
| `public.validation_rules` | Verification constraints for C2B transactions |
| `public.webhook_logs` | Safaricom C2B webhook delivery records |

### ⚠️ Unused / Redundant Tables
Because this dashboard is **single-user only** and will never allow public registrations or multi-tenancy, the following tables are marked as **unused**:
- `public.organizations`: Multi-tenant companies (Redundant).
- `public.users_organizations`: Mapping of users to organizations (Redundant).
- `public.roles`: Dynamic roles permissions (Redundant; admin has all privileges).
- `public.users`: Replaced by `public.me` (or unused profiles).

## 🔒 Row Level Security (RLS)

All tables have RLS enabled to prevent unauthorized reads and writes.

### RLS Policies Configuration
1. **Admin Read/Write Only**:
   - For tables like `transactions`, `mpesa_credentials`, `reversals`, `audit_logs`, and `me`.
   - Policy condition:
     ```sql
     CREATE POLICY "Admin has full access"
     ON public.transactions
     FOR ALL
     TO authenticated
     USING (auth.uid() IN (SELECT id FROM public.me));
     ```
2. **Webhook/API Inserts**:
   - For callback tables (`stk_push_logs`, `webhook_logs`, `transactions`).
   - Policies allow service role or backend API client to insert records without holding an admin session, verified via internal API secrets.
